# 機能仕様書: 振込先口座 ゆうちょ分岐の撤廃（支店名へ統一）

## 1. 機能概要

- **機能名**: `employee-bank-yucho-unify`
- **目的**: 社員の振込先口座入力で、ゆうちょ銀行とそれ以外の銀行で表示を分けている処理を撤廃し、すべて通常銀行と同じ表記（支店名・口座種別・口座番号）に統一する。
- **背景**: 現場運用上、ゆうちょ専用の「記号 / 番号」表記・口座種別の非表示は不要との要望。
- **スコープ（やる）**:
  - 「記号」ラベル → 「支店名」（他行と同一表記）
  - ゆうちょ時に隠していた「口座種別」を常時表示
  - 「番号」ラベル → 「口座番号」、ゆうちょ用 placeholder 撤去
  - 上記を**表示している全箇所**で一貫させる
- **スコープ（やらない）**:
  - DBカラム変更・データ移行（**不要**。両分岐とも同一カラムに保存済み。下記参照）
  - 銀行マスタ（`tenant_payroll_banks`）の変更。ゆうちょ銀行は選択肢として存続
  - 入力値のバリデーション追加

## 2. 影響範囲（実コードで確認済み）

保存先カラムは分岐に依存せず**共通**（`employees.bank_branch_name` / `bank_account_number` / `bank_account_type`、migration 030）。
したがって**マイグレーション・型変更・RLS変更は一切不要**。変更は UI ラベル / 表示条件のみ。

| ファイル | 現状の分岐 | 変更内容 |
|---|---|---|
| `components/employee/ProfileSection1Basic.tsx` | `isYucho`（L40）で 記号/支店名・口座種別非表示・番号/口座番号 を切替（L212-216, 219-230） | `isYucho` 撤去し、非ゆうちょ表記に固定 |
| `app/(admin)/admin/employees/[id]/page.tsx` | `bank_name.includes('ゆうちょ')` で同様に切替（L844-858） | 同上、支店名・口座種別常時・口座番号に固定 |

## 6. 既存機能との差分・依存

- `bank_account_type`（口座種別）は従来ゆうちょ時に非表示。統一後は常時表示。**既存のゆうちょ社員データで `bank_account_type` が NULL のまま**でも、select の「未選択」で表示されるため破綻しない。
- `bank_branch_name` に既にゆうちょの「記号」数字（例: `12345`）が入っている社員は、ラベルが「支店名」に変わるだけで値はそのまま表示される（データ破壊なし）。**運用側で必要なら手入力で支店名へ直す**が、本変更は値を一切書き換えない。

## 7. 実装ルール

- 命名・既存トークン踏襲。`Field` / `EditableRow` の既存呼び出し形を維持し、分岐を平坦化するのみ。
- `any` 不可・`console.log` 残さない（既存方針）。
- ゆうちょ用 placeholder（`'1XXXX'` / `'XXXXXXX'`）は削除（非ゆうちょは placeholder 無し or `'○○支店'`）。
